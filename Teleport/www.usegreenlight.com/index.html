FAQ

How does website version control work?

We keep track of which HTML element a comment is attached to. If you update the website we'll know whether that element still exists or not. We gracefully fallback to show a screenshot if the said element is no longer visible.

Can I use this without an extension?

At the moment we only support Chrome extensions. We found this to be the easiest way to leave comments on websites that have security requirements such as logins, firewalls and proxies.

Why is this better than usepastel?

By design, Greenlight supports commenting on web pages that show up after a user logs in. A significant portion of apps developed today (if not all) have some sort of user access control. Not being able to leave feedback on pages that appear after a user logs in is a deal breaker for us.

Our competitors won't be able to do this gracefully because of how their apps were built.

How can I integrate this with github?

Navigate to your project settings and click connect to GitHub. After you've connected your Greenlight project to your GitHub project, every new comment made will show up as a GitHub issue.

You can also export the current comments from your Greenlight project to GitHub.

How can I integrate this with jira?

Navigate to your project settings and click connect to Jira. After you've connected your Greenlight project to your Jira project, every new comment made will show up as a Jira ticket (or a custom ticket as per your settings).

You can also export the current comments from your Greenlight project to Jira.

How do you handle responsive commenting?

Your comments get automatically tagged by resolution. Users on the project can then easily filter all comments by common device resolution.

If your comment is attached to an element that only appears at certain resolution, we gracefully fallback to showing a screenshot until the attached element reappears.
